01

서비스 개요

유적지마다 담긴 역사 스토리 카드를 드래그해 나만의 하루 여정을 완성하면,
이동 구간별 교통편 · 명소별 맛집 · 날씨가 실시간으로 업데이트되는
경주 전용 여행 플래너
기존 여행 앱의 한계
검색→선택→저장의 단순 반복 구조. 각 명소의 역사적 맥락 없이 위치 정보만 제공되며, 순서를 바꿀 때마다 교통편을 다시 찾아야 하는 번거로움이 있다.
경주 타임트래블의 해법
"스토리로 먼저 흥미를 유발하고, 순서 배치와 동시에 모든 실용 정보가 자동으로 완성된다." 사용자는 이야기를 읽고 고르는 것만으로 완벽한 여정이 완성된다.
02

공모전 요건 부합성

공모 주제 한국관광공사 OpenAPI 필수 활용 — 관광지 정보·운영시간·혼잡도·입장료 실시간 연동
지역 특화 경주시 단일 지역 집중 → 가점 항목 해당 (서울 외 1개 지역 특화 서비스)
대상 플랫폼 웹(PC/모바일 반응형) + 앱 동시 지원 — 공모 부문 웹·앱 개발 직접 부합
주요 이용층 2030 국내 여행자 — 역사 콘텐츠를 감성적으로 소비하는 핵심 세대
03

핵심 UX — 스토리 카드 타임라인

사용자는 세 가지 동작만으로 여정을 완성한다.

①
카드 읽기각 유적지의 역사 일화·비화가 담긴 스토리 카드를 훑어본다
②
카드 배치흥미로운 카드를 오전·오후·저녁 타임라인에 드래그(PC) 또는 탭(모바일)으로 배치한다
③
정보 확인배치 즉시 — 이동 구간별 교통편, 명소별 맛집, 예상 소요시간이 자동으로 완성된다

반응형 인터랙션 설계

환경 PC / 웹 모바일 / 앱
카드 추가 드래그 앤 드롭 탭 → 바텀시트에서 시간대 선택
순서 변경 카드 간 드래그로 원하는 위치에 삽입 롱프레스 후 드래그 (진동 피드백 포함)
레이아웃 3단 분할 (카드풀 · 타임라인 · 맞춤정보) 하단 탭바 (카드선택 · 내여정 · 맞춤정보)

05

실시간 맞춤 정보 시스템

카드 배치가 바뀔 때마다 0.6초 내에 아래 세 가지 정보가 자동 업데이트된다.

🚌 구간별 교통편
"경주역 → 불국사", "불국사 → 석굴암" 등 이동 구간마다 버스·택시·도보 옵션을 각각 표시
순서를 바꿀 때마다 교통편을 따로 검색하던 번거로움 제거
🍽 명소별 맛집
배치된 카드 각각의 주변 맛집을 별도로 추천 (불국사 근처 돌솥밥, 황리단길 한옥 카페 등)
어느 곳을 방문하든 근처 식당 정보가 자동 완성
📊 여정 요약
선택 명소 수, 예상 총 소요 시간, 이동 거리를 즉시 집계
무리한 일정 없이 현실적인 하루 계획 수립 가능
🌤 날씨 안내
경주 당일 기온·미세먼지·자외선 정보와 추천 준비물 표시
계절·날씨에 따른 방문지 재조합 유도
※ 업데이트 응답 속도 안내
맞춤 정보는 카드 배치 변경 후 0.6초 내 갱신을 목표로 설계되었습니다. 다만 한국관광공사 OpenAPI 서버 응답 속도 및 네트워크 환경에 따라 실시간 반영이 다소 지연될 수 있으며, 이 경우 "여정 확정" 버튼을 통해 일괄 조회하는 방식으로 안정적인 정보 제공이 가능합니다. 최종 제출 버전에서는 API 캐싱 전략을 적용하여 체감 응답 속도를 최소화할 예정입니다.
06

차별화 포인트

기존 여행 플래너의 한계
네이버 지도·카카오맵: 위치 검색 중심, 스토리 없음
트리플·마이리얼트립: 찜 목록 방식, 순서 변경 시 정보 미갱신
경주시 공식 앱: 단방향 안내, 사용자 커스텀 불가
경주 타임트래블의 강점
스토리 우선 — 흥미 유발 후 방문 동기 자연 생성
배치 즉시 갱신 — 교통·맛집 정보를 매번 검색할 필요 없음
경주 단일 특화 — 깊이 있는 로컬 콘텐츠 큐레이션

(확장)

4단계
(성장)
내 여정 공유 카드 — SNS 공유용 이미지 자동 생성, 바이럴 유도
장기 목표
경주 외 타 지역 확장 (전주, 안동, 강릉 등) — 전국 역사 여행 플랫폼화
※ 공모전 기간 내 추가 구현 가능성
위 단계별 로드맵은 안정적인 서비스 완성도를 기준으로 수립한 계획이며, 공모전 컨설팅 및 심사 피드백, 팀 역량 집중 여부에 따라 2단계 이상의 기능이 1단계(공모전 제출 버전)에 선반영될 수 있습니다. 특히 카카오맵 실경로 연동 및 스탬프 투어 기능은 개발 우선순위를 높여 조기 적용을 검토 중입니다.
